# Changelog

## v0.1.26

- v0.1.25에서 현재 목록/삭제 목록에 들어가도 아무것도 표시되지 않던 문제를 수정했습니다.
- 원인은 분류 드롭다운 초기화 함수가 동적 분류 상수보다 먼저 실행되면서 JavaScript가 중단되던 초기화 순서 문제였습니다.
- 분류 드롭다운, 분류 관리, 사이트 분류별 보기 기능은 유지했습니다.

## v0.1.25

- 현재 목록/삭제 목록의 분류 변경을 직접 입력 프롬프트가 아니라 드롭다운 선택 방식으로 변경했습니다.
- 선택 항목 일괄 분류 변경도 드롭다운으로만 고르게 했습니다.
- 설정 탭에 분류 관리 영역을 추가해 분류를 추가하거나 이름을 수정할 수 있게 했습니다.
- 설정 탭 사이트 목록에서도 사이트 기본분류를 드롭다운으로 변경할 수 있게 했습니다.
- 설정 탭 사이트 목록을 분류별로 필터링해서 볼 수 있게 했습니다.

## v0.1.24

- 정렬 드롭다운에 `오래된 순`이 반복해서 추가되던 문제를 수정했습니다.
- 정렬 옵션을 `최근 본 순 / 오래된 순 / 제목 순 / 화수 높은 순 / 수정일 최신순 / 수정일 오래된순`으로 정리했습니다.
- 현재 목록과 삭제 목록을 `전체 / 웹툰 / 만화 / 망가 / 소설 / 애니 / 기타` 분류별로 볼 수 있게 했습니다.
- 분류 필터와 페이지당 표시 개수 선택을 브라우저에 저장하도록 했습니다.

## v0.1.23

- 현재 목록/삭제 목록에 페이지네이션 추가
- 기본 100개씩 표시, 50/100/200/500/전체 전환 가능
- 정렬에 오래된 순 추가
- 수정일 최신순/수정일 오래된순도 선택 가능
- 기존 서버 목록 생성 방식은 유지하고, UI 렌더링 단계에서만 페이지 분할

## v0.1.22

- 설정 탭 사이트 목록에서 사이트 표시 이름을 수정할 수 있게 했습니다.
- 설정 탭에서 사이트 기본분류를 바꾸면 현재 목록 표시에도 반영되도록 했습니다.
- 개별 작품에서 직접 수정한 분류는 유지하고, 수동 분류가 아닌 항목만 사이트 기본분류 변경 대상에 포함했습니다.

## v0.1.21

- v0.1.15-startfix 안정판 기준으로 다시 수정했습니다.
- `/novel/작품ID/글ID`, `/manhwa/작품ID/slug` 상세 주소를 기록하되 URL 뒤 숫자/slug는 화수로 쓰지 않게 했습니다.
- 제목에 `17화`, `17장`, `17회`, `17편`, `챕터 17`, `Chapter 17`, `ch.17`처럼 명확한 표식이 있을 때만 화수를 기록합니다.
- 화수가 없으면 작품은 기록하고 화수 칸은 비워둡니다.
- `오해 (2)` 같은 괄호 숫자와 부제목은 화수로 쓰지 않습니다.
- 숫자 변경 도메인 계열의 fallback 판단을 `sbxh`에만 묶지 않고 동적 도메인 규칙에 맞춰 적용했습니다.
- 현재/삭제 목록 버튼 줄을 `열기 / 화수선택 / 분류 / 제목수정 / 삭제·복구·완전삭제` 순서로 보정했습니다.

## v0.1.15

- 백그라운드 실행이 반복되거나 잘못된 포트를 여는 문제를 수정했습니다.
- 실제 서버 포트가 확인될 때만 브라우저를 열도록 변경했습니다.
- 서버 종료 스크립트가 시작 대기 중인 PowerShell 프로세스까지 정리하도록 보강했습니다.

## v0.1.14

- 0.1.14은 v0.1.13 안정판을 기준으로 정리했습니다.
